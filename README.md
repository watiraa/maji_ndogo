# Restoring the Flow of Maji Ndogo & Rebuilding Hope

## Introduction

This project addresses the **water crisis** in a fictional country named **Maji Ndogo**. It explores the complexities of leveraging SQL to tackle intricate data problems, extracting intriguing and thought-provoking insights along the way.

## Acknowledgments

This project was completed as part of the requirements for the Data Science curriculum at [ALX + Explore AI Academy](https://www.explore.ai/learn/courses/data-science)

## Project Overview

The project is divided into four parts, each one progressively increasing in SQL syntax complexity and analytical depth. It serves as a comprehensive guide on how to approach and resolve data-driven challenges.

# Part 1: Beginning Our Data-Driven Journey in Maji Ndogo

In the first part of the project, we delve into Maji Ndogo's extensive database, which contains 60,000 records across various tables. We will navigate this rich dataset using basic queries to become familiar with the content of each table while also refining some of the data through Data Manipulation Language (DML).

### Objectives

By the end of the first quadrant of the project, one should be able to:

- Demonstrate the ability to **create** and **navigate through a new database**.
- Demonstrate the ability to **create** and **modify basic queries**.
- Demonstrate the ability to use operators to create conditional filters.
- Illustrate the **use of operators for creating conditional filters**.
- Demonstrate the **use of DML** and discuss the **risks involved in altering a database**.


# Part 2: Clustering Data to Unveil Maji Ndogo's Water Crisis

In this second part of the project, we prepare for an in-depth analytical exploration of Maji Ndogo's water situation. We will leverage a diverse array of **functions**, including **complex window functions**, to extract valuable insights from the data tables.

### Objectives

By the end of the second quadrant of the project, one should be able to:

- Gain an understanding of various **SQL functions**.
- Demonstrate an understanding of how **SQL window functions** work and how to use them effectively.
- Demonstrate the ability to **use various functions to clean and analyse data** efficiently.
- Demonstrate an understanding of **data aggregation techniques**.

# Part 3: Weaving the Data Threads of Maji Ndogo's Narrative

In the third part of the project, we will extract data from **multiple tables** and perform **statistical analyses** to evaluate the implications of an audit report that cross-references a random sample of records.

### Objectives

By the end of the third quadrant of the project, one should be able to:

- Demonstrate the ability to retrieve data from various sources.
- Demonstrate an understanding of the challenges associated with joining multiple data sources together.

# Part 4: Charting the Course for Maji Ndogo's Water Future

In the final part of the project, we will conclude our data analysis using the **full suite of SQL tools**. We will derive our final insights, categorize water sources, and prepare relevant information for relevant stakeholders i.e. Maji Ndogo's engineering teams.

### Objectives

By the end of the final quadrant of the project, one should be able to:

- Demonstrate the ability to **integrate and utilise filters, aggregations, functions, and joins together** in data analysis.
- Demonstrate an understanding the **complexities involved in addressing real-world problems using SQL**.
- Demonstrate the ability to **utilize subqueries, Common Table Expressions (CTEs), and views** effectively.
